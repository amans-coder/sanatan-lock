## 1. IPC Pattern (TypeScript ↔ Kotlin)

**Decision:** Module-first IPC. SharedPreferences for crash recovery ONLY.

```
TS ──method call──▶ SadhanaBlockerModule.kt ──bindService()──▶ AccessibilityService
Kotlin ──sendEvent()──▶ Expo EventEmitter ──▶ TS listeners

SharedPreferences: written by module on every state change.
                   Read by service ONLY on cold start (after process death).
                   NEVER the primary communication channel at runtime.
```

**Why:** SharedPreferences has no cross-process change notification. On Xiaomi/Samsung,
the AccessibilityService may run in a separate process and never see TS-side writes.
The Expo Module bridge provides direct method calls that are process-safe.

**Implementation:**
- `SadhanaBlockerModule.kt` binds to `SadhanaAccessibilityService` via `ServiceConnection`
- All `set*()` methods go through the bound service directly
- On every state mutation, the module ALSO writes to SharedPreferences (backup)
- On service cold start, service reads SharedPreferences to restore last-known state

               └───────────┘

### All decisions RESOLVED:

- **Night lock timing:** **Configurable** by user via Settings time pickers. Default: 10 PM to 6 AM. **Device local timezone.**
- **Morning gate definition:** User must open the SadhanaLock app and pray on the prayer screen. Overlay dismissal alone does NOT count.
- **Cooldown duration:** Configurable (5/15/30/60 min), default 15 min. User sets in Settings.
- **Gentle vs Strict mode:**
  - **Gentle:** See mantra, tap Om immediately (2-3 seconds). No forced wait.
  - **Strict:** See mantra, forced wait 10 seconds, then Om button appears, tap to dismiss.
- **Clock manipulation protection:** Not required. Trust the user. Use device system time directly.

## 3. App Icon Loading Strategy

**RESOLVED — Option B: Paginated (on-demand per app).**

`getInstalledApps()` returns the app list **without icons** (lightweight). A separate `getAppIcon(packageName): Promise<string>` method fetches a single app icon on demand (base64). The UI lazy-loads icons as list items scroll into view.

**Why:** Avoids sending 4-10MB through the bridge at once. No ANR risk on low-end devices.

**Contract change needed:** Add `getAppIcon(packageName: string): Promise<string | null>` to the module contract.

## 4. Process Model

**RESOLVED — Same-process assumption, with detection fallback.**

By default, Android AccessibilityService runs in the app's main process. But some OEMs
(Xiaomi MIUI, Huawei EMUI) may fork it into a separate process for "optimization."

**If same process:** Module can bind directly, SharedPreferences work normally.
**If separate process:** Need `ContentProvider` or `Messenger`/`AIDL` for IPC. SharedPreferences
writes from main process are invisible to service process without `MODE_MULTI_PROCESS` (deprecated).

**Recommendation:** Start with same-process assumption. Add `android:process` check in
HealthCheckWorker. If separate process detected, log a Crashlytics event and fall back
to `ContentProvider`-based IPC.

## 5. Permission Denial Edge Cases

**RESOLVED — Follow the table below.**

| Permission | 1st denial | 2nd denial | 3rd+ denial |
|---|---|---|---|
| Accessibility | Re-explain → retry | Re-explain → retry | Android blocks re-asking. Must deep-link to Settings. Show permanent banner. |
| Overlay (SYSTEM_ALERT_WINDOW) | Re-explain → retry | Settings deep-link | Settings deep-link |
| Battery optimization | Non-blocking warning | Non-blocking warning | Accept — service may be killed more often |
| QUERY_ALL_PACKAGES | N/A (declared in manifest, granted at install on A10-) | N/A | On A11+: must be declared. If Play Store strips it, `getInstalledApps()` returns partial list. |

## 6. Bypass Protection

**RESOLVED — Implement all mitigations in the table below.**

| Bypass vector | Risk | Mitigation |
|---|---|---|
| Swipe away via Recent Apps | HIGH | Use `TYPE_ACCESSIBILITY_OVERLAY` (not `TYPE_APPLICATION_OVERLAY`). This renders above Recent Apps. |
| Force-stop SadhanaLock | HIGH | HealthCheckWorker detects and shows notification. Can't fully prevent — user is in control. |
| Uninstall SadhanaLock | LOW | Can't prevent. App should be valuable enough that user doesn't want to. |
| Disable Accessibility in Settings | MEDIUM | HealthCheckWorker detects within periodic check interval. Show persistent notification. |
| Phone call during overlay | MEDIUM | Overlay must detect `CALL_STATE_RINGING` and auto-dismiss. Register `PhoneStateListener`. |
| Screen rotation during overlay | LOW | Lock overlay to portrait orientation. |

## 7. Protected Apps Denylist

**The following packages MUST NOT be blockable:**

```kotlin
val PROTECTED_PACKAGES = setOf(
    "com.sadhanalock.app",          // self
    "com.android.systemui",         // system UI
    "com.android.launcher",         // default launcher (varies by OEM)
    "com.android.launcher3",
    "com.google.android.apps.nexuslauncher",
    "com.sec.android.app.launcher", // Samsung
    "com.miui.home",                // Xiaomi
    "com.android.dialer",           // phone
    "com.android.contacts",         // contacts
    "com.android.settings",         // settings
    "com.android.emergency",        // emergency
)
// Also: any app with category LEANBACK_LAUNCHER or HOME
```

**RESOLVED:** User **CANNOT** override the denylist. Hard-coded, no Settings toggle.

## 8. SharedPreferences Schema (Updated)

Single file: `sadhana_blocker_prefs`. Used for **persistence only** (runtime state goes through module bridge).

| Key | Type | Writer | Reader | Description |
|---|---|---|---|---|
| `blocked_packages` | `Set<String>` | Module | Service (cold start) | Apps to intercept |
| `mode` | `String` | Module | Service (cold start) | `gentle` / `strict` |
| `selected_deity` | `String` | Module | Service (cold start) | Deity for overlay |
| `overlay_content_json` | `String` | Module | Service (cold start) | Prayer content |
| `service_enabled` | `Boolean` | Module | Service (cold start) | Master kill switch |
| `last_prayer_ts` | `Long` | Service | Module | Last prayer timestamp |
| `prayer_count_today` | `Int` | Service | Module | Daily stats |
| `night_lock_enabled` | `Boolean` | Module | ScheduleManager | Night lock toggle |
| `night_lock_start_hour` | `Int` | Module | ScheduleManager | e.g., 22 (10 PM) |
| `night_lock_end_hour` | `Int` | Module | ScheduleManager | e.g., 6 (6 AM) |
| `morning_prayer_done_today` | `Boolean` | Service | ScheduleManager | Reset at night_lock_end |
| `cooldown_minutes` | `Int` | Module | Service | Unlock window duration |
| `detection_method` | `String` | Module | Service | `accessibility` / `usage_stats` |

**DO NOT store deity images here.** Use file storage + URI.

## 9. Deity Image Storage

**Decision:** File-based, NOT SharedPreferences.

```
/data/data/com.sadhanalock.app/files/deity_images/
  ├── krishna.png
  ├── shiva.png
  ├── hanuman.png
  ├── lakshmi.png
  ├── ganesh.png
  ├── durga.png
  └── saraswati.png
```

Expo bundles these in assets. On first launch, the module copies them to internal storage.
SharedPreferences stores only the file path: `deity_image_path` → `/data/.../krishna.png`.
OverlayService loads from this path.

## 10. Firebase Crashlytics Integration

**Scope: Crashlytics ONLY.** No Firebase Auth, Firestore, Analytics, or other services.

**Setup:**
1. Add `google-services.json` to `/frontend/android/app/`
2. Add Crashlytics Gradle plugin to `build.gradle`
3. Add `@react-native-firebase/crashlytics` to `package.json`
4. Custom crash keys: `device_oem`, `detection_method`, `overlay_state`, `service_uptime`
5. Custom events: `service_killed`, `service_restarted`, `overlay_shown`, `prayer_completed`

## 11. Critical Failure Modes (Must Have Tests)

| # | Failure | What happens | Test type |
|---|---|---|---|
| 1 | OEM kills AccessibilityService | HealthCheckWorker detects, shows notification | Instrumented |
| 2 | Overlay permission revoked mid-session | `addView()` throws, catch → show notification | Unit |
| 3 | Doze mode defers Night Lock alarm | Use `setExactAndAllowWhileIdle()` | Manual on device |
| 4 | Recent Apps swipe bypasses overlay | Use `TYPE_ACCESSIBILITY_OVERLAY` flag | Manual on device |
| 5 | Double-tap Om button | Debounce: ignore taps within 500ms of first | Unit |
| 6 | Morning gate flag lost after force-stop | Write to SharedPreferences with `commit()` (sync) | Instrumented |
| 7 | 200+ apps in getInstalledApps() | File-based icons, lazy load | Performance |
| 8 | User blocks their launcher | Denylist check before persisting | Unit |

## 12. Resolved Questions (All decided 2026-05-28)

1. **Night lock timing:** Configurable, device local timezone. Default 10 PM to 6 AM.
2. **Cooldown duration:** Configurable (5/15/30/60 min), default 15 min.
3. **Gentle vs Strict:** Gentle = instant Om tap. Strict = 10s forced wait then Om.
4. **Morning gate:** Must open SadhanaLock app and pray on prayer screen (overlay does not count).
5. **Clock manipulation:** Not required. Trust the user.
6. **Denylist overridable:** NO. Hard-coded, user cannot override.
7. **Icon loading:** Paginated. getAppIcon(pkg) on demand, no bulk transfer.
8. **UsageStats polling:** Detection latency acceptable. Use battery-friendly interval (~1500-2000ms).
